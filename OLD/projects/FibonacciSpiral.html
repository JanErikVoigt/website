The Code

The majority of the sourcecode is not that interesting. You can find the 'essence' of this project in the function below.

One interesting thing you might notice is the pow-function around the distance-calculation. You can actually tweak this parameter in the demo. You might notice power=1 (which is the same as leaving the pow-function out) results in a very dense spiral in the middle. If you instead choose 0.5 (which makes it a square-root), the points are very nicely spread out. This is also an idea I got from Sebastian Lague.

function generateFibonacciVertices(num_points, power = 0.5) {
    let verts = []
    var turn_fraction = 1.6180339 // golden ratio

    for(let i = 0; i < num_points;i++) {
        var distance = pow(i / (num_points - 1.0), power)
        var angle = 2 * PI * turn_fraction * i
            
        var x = distance * cos(angle)
        var y = distance * sin(angle)
        
        verts[i] = createVector(x,y)
    }
    return verts
}
